# mnemom/.github

Organization-wide community-health files and defaults for the
[mnemom](https://github.com/mnemom) GitHub organization.

GitHub applies files in this repository as the **default** for every repo in
the organization that does not provide its own. See GitHub's
[community health files](https://docs.github.com/en/communities/setting-up-your-project-for-healthy-contributions/creating-a-default-community-health-file)
documentation for how defaults are resolved.

## What lives here

| File | Purpose |
| --- | --- |
| [`profile/README.md`](profile/README.md) | The organization profile shown at [github.com/mnemom](https://github.com/mnemom). |
| [`LICENSE`](LICENSE) | Default license (Apache-2.0) for repos without their own. |
| [`SECURITY.md`](SECURITY.md) | How to report a security vulnerability. |
| [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md) | Contributor Covenant, applied org-wide. |
| [`CONTRIBUTING.md`](CONTRIBUTING.md) | Branch, commit, and PR conventions. |
| [`.github/ISSUE_TEMPLATE/`](.github/ISSUE_TEMPLATE) | Default bug-report and feature-request issue forms. |
| [`.github/PULL_REQUEST_TEMPLATE.md`](.github/PULL_REQUEST_TEMPLATE.md) | Default pull-request template. |

A repository that ships its own copy of any of these files overrides the
default here for that repository.

## Reporting a security issue

Please follow [`SECURITY.md`](SECURITY.md) — do not open a public issue for
security vulnerabilities.
